Architecture that matters

The v1 suite demonstrates multiple Pentaho surfaces working together: repository content, managed JDBC, CDA, CDF, Analyzer artifacts, Mondrian cubes, reusable build tooling, and versioned deployment.

Reusable pattern

Each iteration is a complete suite published to its own repository folder, allowing v1, v2, and future releases to coexist without breaking demos or buyers' validation paths.

Open analytics surface

CDA exposes data as queryable services, CDF provides native dashboard runtime capability, and dependency-free HTML/SVG dashboards make demos reliable in locked-down environments.

Governed enterprise spine

Managed JNDI connections and modeled warehouse facts keep the analytics grounded in Pentaho-managed infrastructure rather than hard-coded extracts.

Why this is a platform story

The strongest message for technical leaders is breadth plus composability. Pentaho is not only rendering reports; it is orchestrating repeatable analytics over enterprise metadata and operational signals.

CDA services

Data APIs without app sprawl

CDA descriptors expose reusable query endpoints over managed data connections, so the same governed facts can power command centers, CDF dashboards, custom HTML, and automation checks.

CDF runtime

Native dashboard composition

The suite includes a true CDF dashboard using the Pentaho CDF runtime, CCC chart components, CDF filters, and tables over CDA queries, proving native extensibility alongside custom dashboard experiences.

Mondrian and Analyzer

Semantic model continuity

Analyzer and .xdash artifacts sit beside the HTML/CDA dashboards, preserving semantic-model workflows while allowing richer technical demos where Analyzer is available.

Lineage facts

Operational control plane

Lineage is treated as an operating dataset: success rates, runtime, cross-boundary flow, sensitive movement, top producers, top consumers, and blast-radius style questions.

Governance model

Risk becomes routeable

Policy assignments, sensitivity, privacy terms, ownership, freshness, and completeness roll into dashboards that support remediation, audit prep, and accountable stewardship.

Versioned delivery

Demoable and repeatable

The suite is built as a self-contained iteration with deploy scripts, stamped repository paths, and local verification through Pentaho APIs, making it easier to demo, extend, compare, and preserve.

The technical leader brief

Pentaho's power is that it can meet the enterprise where the data already lives, model the operational and governance facts around that estate, and publish decision-grade analytics through multiple native and open surfaces.

This v1 suite shows the connective tissue: repository deployment, managed connections, CDA query services, CDF dashboards, HTML/SVG experiences, Analyzer-ready artifacts, and versioned iteration. That is more than BI. It is a platform for data product observability.

For CIOs and CDOs

The platform connects technical metadata to business-control questions: where risk lives, where spend accumulates, where data quality blocks trust, and which integration flows deserve operational attention.

For enterprise architects

The implementation proves composability. CDA, CDF, Mondrian, Analyzer, repository content, and custom HTML can coexist around the same governed facts instead of forcing every use case into one presentation layer.

For platform owners

The suite is operationally practical: dependency-free dashboards, explicit managed datasource requirements, cacheable query services, version-stamped repository paths, and repeatable deployment scripts.

For data governance leaders

The dashboards turn vague governance language into counts, percentages, rankings, and flows: 12,565 ungoverned assets, 625 privacy-related assets, 161 terms, 305 policy assignments, and measurable owner gaps.
